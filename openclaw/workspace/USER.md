## How This Agent Is Used

This agent is deployed per-client. Each deployment serves one client company's onboarding needs. The operator configures:

1. `config/client.js` — brand, persona, qualification rules, intake fields, FAQ
2. `.env` — API keys, CRM credentials, webhook URLs
3. Channel connections — which messaging platforms to activate

## Preferences

- Respond to all inbound messages on connected channels
- Keep conversations focused on onboarding — redirect off-topic requests
- Escalate complex questions to humans rather than guessing
- Log all lead submissions and escalations
- Never share data between different users or channels
